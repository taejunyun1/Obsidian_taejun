# Obsidian Vault Rules

이 프로젝트는 Obsidian Vault이다.

## 조직 원칙

이 Vault는 Tiago Forte의 PARA Method를 변형해서 사용한다.

- 00_INBOX: 아직 분류되지 않은 임시 노트
- 10_PROJECTS: 마감, 완료 기준, 결과물이 있는 현재 진행 작업
- 20_AREAS: 장기적으로 유지해야 하는 책임 영역
- 30_RESOURCES: 미래에 참고할 주제 자료
- 40_ARCHIVES: 완료되었거나 비활성화된 자료
- 50_MAPS: MOC, 인덱스, 주제 지도 노트
- 90_SYSTEM: 템플릿, 첨부파일, Codex 로그, 스크립트

## 분류 규칙

1. 프로젝트는 반드시 완료 기준이 있어야 한다.
2. 책임 영역은 완료되지 않고 지속 관리된다.
3. 리소스는 특정 마감이 없는 참고자료이다.
4. 아카이브는 현재 비활성 자료이다.
5. 주제만 보고 폴더를 정하지 말고, 현재성/실행성 기준으로 정한다.
6. 확신이 낮은 파일은 이동하지 말고 review-needed로 표시한다.
7. 학생 개인정보, 평가자료, 민감 자료는 이동하거나 GitHub에 포함하기 전에 사용자 확인을 요구한다.

## Obsidian 링크 규칙

1. 내부링크는 `[[노트 제목]]` 형식을 사용한다.
2. 폴더 이동만으로 의미 연결을 대체하지 않는다.
3. 의미 연결은 MOC 또는 내부링크로 만든다.
4. 본문을 임의로 재작성하지 않는다.
5. 링크 추가는 별도 작업으로 수행한다.

## Codex 작업 규칙

1. 대량 이동 전에는 항상 migration map을 먼저 만든다.
2. 승인 없이 파일을 삭제하지 않는다.
3. 승인 없이 파일명을 바꾸지 않는다.
4. 이동 후 변경 내역을 로그로 남긴다.
