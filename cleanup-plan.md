# Vault Cleanup Plan

작성일: 2026-06-22

## 범위

- 대상 Vault: `Imac_Taejun`
- 검사 대상: Markdown 파일 37개
- `AGENTS.md`: 발견되지 않음
- 이번 단계에서 기존 노트 본문은 수정하지 않음
- 이번 단계에서 만든 파일: `cleanup-plan.md`, `cleanup-report.md`

## 정리 원칙

- 문장 재작성 금지
- 의미 요약, 보강, 삭제 금지
- 원본 노트의 내용 추가/삭제 금지
- 파일 이동, 이름 변경, 병합, 삭제 금지
- 내부 링크 대상 추측 수정 금지
- 애매하면 수정하지 않고 보고서에 기록

## 1단계: 안전 점검

1. 작업 전 `git status --short`로 기존 변경사항을 확인한다.
2. 이전 단계에서 이미 변경된 노트와 새로 정리할 노트를 구분한다.
3. 자동 정리는 한 번에 전체 적용하지 않고, 항목별로 나누어 적용한다.
4. 적용 뒤에는 내부 링크 검사와 `git diff --check`를 다시 실행한다.

## 2단계: 자동 수정해도 비교적 안전한 항목

아래 항목은 문장 의미를 바꾸지 않는 기계적 정리 후보이다. 그래도 적용 전후 diff 확인은 필요하다.

- 줄 끝 공백 제거
- 연속 빈 줄을 과도하게 늘린 곳 정리
- 파일 끝 개행 정리
- 명백한 비출력 제어문자 제거 후보 표시
- 시스템 보고서 파일의 단순 헤딩 공백 정리

주의: 리스트 기호 바로 뒤 공백 추가, 헤딩 레벨 변경, `---` 구분선 변경은 렌더링 구조를 바꿀 수 있으므로 자동 수정 대상에서 제외한다.

## 3단계: 사람 확인 후 처리할 항목

- `-텍스트`처럼 리스트인지 문장부호인지 애매한 줄
- `####` 또는 `###`로 시작하는 노트의 헤딩 레벨
- 본문 중간에 반복되는 `---` 구분선
- HTML 헤딩을 Markdown 헤딩으로 바꿀지 여부
- 이미지 임베드의 `\|숫자` 표기
- 빈 노트, 거의 같은 제목의 노트, 날짜만 다른 회의/계획 노트
- inline `#태그`를 YAML frontmatter로 옮길지 여부

## 4단계: 절대 자동 수정하지 않을 항목

- 노트 제목 변경
- 폴더 이동
- 유사 노트 병합
- 빈 노트 삭제
- 본문 문장 재작성
- 작품명, 작가명, 이론명, 전시명 표기 통일
- 깨진 링크의 대상 추측
- 새로운 내부 링크 대량 삽입
- alias/frontmatter 신규 생성

## 폴더 구조 제안

현재 구조는 이미 큰 축이 나뉘어 있다. 보수적으로는 새 폴더를 만들거나 파일을 옮기기보다, 다음 방향만 후보로 둔다.

- `00_SYSTEM`: 분석 보고서, 링크 제안, 트러블슈팅 문서 유지
- `Work/Transmitter to Object`: 해당 프로젝트의 촬영, 기술, 텍스트 계획 유지
- `Work/Research`: 논문과 연구 계획 유지
- `Work/Residence`: 레지던시/지원 관련 문서 유지
- `전시관련`: 전시, 토크, 기관 접촉 관련 문서 유지
- `Arkive`: 과거 작업, 질문, 임시 보관 문서 유지
- `프로토콜 콜렉티브`: 날짜별 회의/작업 기록 유지

추후 사람이 확인한 뒤에만 검토할 후보:

- `00_SYSTEM` 내부에 `Reports` 하위 폴더를 둘지 여부
- `Arkive` 내부의 빈 노트 또는 임시 노트 처리 방식
- `전시관련`과 `Work/Residence` 사이의 기관/전시 문서 경계
- `Work/Transmitter to Object/Shot_Plan`의 연도별 계획 분리 여부

## 적용 순서 제안

1. 기존 git 변경사항을 먼저 커밋하거나 별도 확인한다.
2. 줄 끝 공백, 파일 끝 개행, 과도한 빈 줄만 별도 커밋으로 정리한다.
3. 제어문자 후보 2곳은 사람이 해당 줄을 확인한 뒤 제거 여부를 결정한다.
4. 헤딩/리스트 문법 후보는 파일별로 diff를 보며 수동 처리한다.
5. 이미지 임베드 `\|숫자` 후보는 Obsidian에서 렌더링을 확인한 뒤 수정 여부를 결정한다.
6. 중복 후보는 삭제/병합하지 않고, 필요하면 `00_SYSTEM`에 별도 검토 목록을 남긴다.
