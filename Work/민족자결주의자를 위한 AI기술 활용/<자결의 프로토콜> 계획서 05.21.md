**〈자결의 프로토콜〉 구현 계획 정리 (업데이트 버전)**
**1. 작업 개요**
〈자결의 프로토콜〉은 정치적 수사와 텍스트와 관련된 이미지 시스템의 관계를 다루는 설치 작업이다.  
작업은 ‘자결’, ‘독립’, ‘주권’, ‘해방’, ‘민족’, ‘국가’와 같은 정치적 언어가 웹 데이터와 플랫폼 환경 속에서 지속적으로 수집·분석·재조합되는 과정을 기반으로 한다.

수집된 정치적 언어는 AI 시스템을 통해 하나의 선언문 형태로 재구성되며, 이 선언문은 다시 생성형 이미지의 프롬프트가 된다. 생성된 이미지와 텍스트는 LED 스크린에 지속적으로 표시되고, 이후 A3 수용지로 출력되어 파쇄와 융해 과정을 거쳐 물질적으로 소멸된다.

이 작업의 핵심은 정치적 이미지를 단순히 보여주는 것이 아니라, 정치적 언어가 플랫폼 환경 안에서 끊임없이 생성·증폭·폐기되는 순환 구조 자체를 드러내는 데 있다.

웹 크롤링을 통해:
- 뉴스 기사
- 정치 연설
- 선언문
- 댓글

- 국제 정치 담론

- 수집하게될 뉴스의 기간과 위치 등

↓

생성 이미지는 직접적인 정치 포스터보다:

- 깨진 프로파간다

- 데이터 노이즈
- 생성 실패한 정치 이미지

등의 형태에 가깝게 구성된다.

이미지는 선언문의 시각적 부산물처럼 작동한다.

**4. 설치 구성**

**A. 중앙 장치: 생성·출력·폐기 시스템**
전시장 중앙에는 복합기, 컴퓨터, 케이블, 출력물, 스크린  함께 배치된다.

이 구조는 하나의:

- 정치 이미지 생산 공장
- 데이터 센터
- 프로파간다 서버룸
- 자동화된 선언 시스템

처럼 보이도록 구성한다.

구성 요소:
- 구형 복합기 또는 A3ㅡ프린터
- 슈퍼미러 위에 녹아져있는 종이들 (사진)
- LED 스크린
- 제어용 컴퓨터
- 멀티탭, 전원 장치
- 케이블 및 네트워크 배선

프린터와 배선은 숨기지 않고 노출한다.  
작업은 완성된 미디어 장치보다, 지속적으로 작동하는 산업적 시스템처럼 보이는 것이 중요하다.
⸻
**B. LED 스크린: 시스템 인터페이스**
LED 스크린은 단순한 디스플레이가 아니라 시스템의 내부 상태를 보여주는 인터페이스다.

- 정치 키워드 스트림
- 선언문 fragment
	- 텍스트 흘러 내리는 방식으로.
- 시스템 로그 (작은 스크린)
	- 오류 메시지
	- 생성 상태

LED는 고해상도보다:
- 산업용 전광판
- 서버 로그
- 공항 상태창
- 증권 시세판

같은 저해상도 인터페이스 느낌이 적절하다.
- 픽셀피치 2~2.5
⸻
**C. 파쇄와 수조: 이미지의 물질적 붕괴** (선택적)
출력된 이미지는 일정 시간 후 파쇄되고, 파쇄된 조각은 수조 안으로 떨어진다.
수용지는 물 안에서 녹으며 정치적 이미지가 물질적으로 붕괴된다.
중요한 것은 “완전 자동화”보다:
생성 → 출력 → 폐기 → 융해

라는 순환 구조 자체다.

현실적 구현 권장안:
- 출력은 자동화
- 파쇄는 반자동
- 일정 시간마다 스태프 개입 가능
- 파쇄기는 수조 위에 설치
- 완전 자동 파쇄는 1차 구현에서 선택사항

이 작업에서 인간 개입은 오히려 시스템 유지관리 퍼포먼스처럼 작동할 수 있다.
⸻

**8. 최종 구현 방향**

이번 버전의 핵심은 완벽한 자동화가 아니라:

정치적 언어가 선언문으로 생성되고,  
선언문이 이미지로 변환되며,  
이미지가 출력·파쇄·융해·저장되는 순환 구조를  
실제 공간 안에서 지속적으로 작동시키는 것

이다.

우선순위는 다음과 같다.

1. 정치 선언문 생성 시스템 구축
2. LED를 통한 선언문 + 로그 + 이미지 표시
3. 자동 이미지 생성 및 저장
4. 자동 출력
5. 파쇄 및 수조 융해
6. CCTV 생중계
7. 완전 자동 파쇄는 선택적 구현

이 구조라면 현재 예산과 기술 수준 안에서 충분히 구현 가능하며, 작업의 개념적 밀도 또한 유지할 수 있다.
